hexo+github博客搭建

Posted on 2018-05-24 | In 工具使用

电脑系统win10,64位

前提条件：

Git、node.js(新版的node.js安装好后自带npm，无需再安装npm)、Github账号

步骤：

1.安装node.js

（1）下载地点：https://nodejs.org/en/download/ ，按照指示一直next安装即可，根据自己需求选择安装位置18:51

（2）安装完成后检查node版本，cmd进入命令行，输入：node –version

检查npm是否安装，输入:npm –verison

2.安装配置Git

（1）windows下安装包：https://gitforwindows.org/ ，根据指示安装

（2）安装成功后，鼠标右键会出现：git bash here

3.安装Hexo

选择合适的地方新建一个文件夹，用来安装Hexo，以后的博客也是放在该文件夹下的，我是选择在E盘下新建的一个空文件夹（E:\blog）。文件夹建好后，通过命令行进行到E:\blog

（1）安装Hexo，输入：npm install hexo -g

完成后，检查Hexo是否安装成功，输入：hexo –version

（2）初始化该文件夹，输入：hexo init ，忘了截图，最后会显示：

INFO Start blogging with Hexo!

说明初始化成功

（3）安装所需组件，命令行输入：npm install

（4）生成部署文件，启动本地服务

命令行输入：hexo g

开启服务器，命令行输入：hexo s

输入该网址看一下结果，发现出现了问题

原因是我的4000端口已经被占用了，所以需要重新给它分配一个端口，因此，将5000端口分配给它。

在命令行输入：hexo server -p 5000

成功

4.github新建仓库

（1）没有github账号的先在https://github.com/注册一个账号

（2）新建一个仓库

新建一个仓库，仓库名必须是：.github.io ,如图所示

5.将Hexo和Github 联系起来

（1）设置Git的user name 和 email，生成密钥

鼠标右键——>git bash here

设置name和email，在git bash中输入：

git config –global user.name “你的名字，可以随意取”

git config –global user.email “你的邮箱号码，例如111@qq.com“

生成ssh密钥，在gitbash中输入：ssh-keygen -t rsa -C “你注册github时候的邮箱号码，例如111@qq.com”

我没有设置密码，就是一直回车即可，如果有设置密码要记住密码。

以上步骤完成后，查看是否有生成id-rsa和id-rsa.pub文件（默认存储路径：C:\Users\User.ssh）：

输入：cd ~/.ssh

输入：ls

（2）将密钥添加到ssh-agent ，在git bash中输入：eval “$(ssh-agent -s)”

将生成的SSH key添加到ssh-agent：在git bash中输入：ssh-add ~/.ssh/id_rsa

（3）在github中添加SSH

登录github，进入setting

进入“SSH and GPG keys”后选择“New SSH key”

将id_rsa.pub 中的所有内容复制进去

测试ssh是否添加成功：在git bash中输入：ssh -T git@github.com

出现了 “Hi 你自己的用户名“就说明添加成功

6.修改_config.yml文件

修改deploy（文件末尾）

将deploy下的内容修改为：

type： git

repository: 步骤4中所建仓库（.github.io）的ssh

所复制的ssh就是需要复制到repository中的SSH

（在后续的hexo d -g 是出现命令一直不能完成的问题，解决方案：1.deploy下的所有key和value之间有空格； 2. 将repository后的SSH中 “http://github/”改为"git@github.com:"）

7.新建博客

在命令行输入命令：hexo new post “博客名称” （命令行在目录E:\blog下）

创建成功，博客保存在“E:\blog\source\images”中，进入文件夹发现已新增文件“newBlog.md”

对其进行编辑，我使用的文本编辑工具是typora

将博客发布前，安装hexo-deployer-git自动部署发布工具 ，在git bash输入命令：

npm instal lhexo-deployer-git –save

进行生成及部署。在git bash输入命令：hexo d -g

完成

8.查看博客

输入网址https://你的用户名.github.io

9.“图片加载不出来”解决方案

解决方案参考：https://blog.csdn.net/Sugar_Rainbow/article/details/57415705

（1）将配置文件_config.yml 里的post_asset_folder:这个选项设置为true

（2）在E:\blog下执行指令：npm install hexo-asset-image --save

（3）在博客中想引入图片时，确定该博客（A.md）的目录下有一个同名的文件夹用于存放所需图片

（4） 最后在A.md中想引入图片时，先把图片复制到A这个文件夹中，然后只需要在A.md中按照markdown的格式引入图片：

![](A/图片名.jpg)

（5）以Typora为编辑器为例说明：

a. 打开Typora中文件——偏好设置

b.在图片插入处进行设置，我选择的最后一项，这样不同博客之间的图片不会错乱

c.设置成功后，通过ctrl+V粘贴到Typora中的图片会自动保存到相应文件夹中，并且会自动生成如下md格式

d.将A.assets（例子中是hexo-github博客搭建.assets）中的所有图片复制到A（例子中是hexo-github博客搭建）文件夹中，并将“/../hexo-github博客搭建.assets/”改为“hexo-github博客搭建”

e.图片出现啦

10.Hexo风格选择

在官网https://hexo.io/themes/选择自己喜欢的风格
